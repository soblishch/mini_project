### Підрахунок Profit/Lost

Ми створили список для зберігання прибутку та втрат кожного дня:
``profit_lost = []``

Наш прибуток та втрати це різниця ціни Close поточного дня та попереднього.
Ми проходимося по всій таблиці перевіряючи сигнали: 

#### так як у першого дня немає попереднього, то записуємо просто 0
```text
    if i == 0:
        profit_lost.append(0)
        continue
```
#### Зміна з 0 на 1, то ми купуємо акцію (у день купівлі прибуток ще 0)
```text
    if data["Signal"].iloc[i] == 1 and data["Signal"].iloc[i-1] == 0:
        profit_lost.append(0)
``` 
#### якщо є послідовність з 1, то ми тримаємо акцію (прибуток це різниця)
```text
        profit = data["Close"].iloc[i] - data["Close"].iloc[i - 1]
        profit_lost.append(profit)
``` 
#### якщо зміна з 1 на 0, то ми продаємо акцію (прибуток це різниця)
```text
        profit = data["Close"].iloc[i] - data["Close"].iloc[i - 1]
        profit_lost.append(profit)
```  
#### якщо зміна з 0 на 0, то акцію не купуємо (прибуток 0)
``    else:
        profit_lost.append(0)``

*iloc* - команда, що витягує рядок у Pandas 

Формуємо нову колонку в таблиці: ``data["Profit/Lost"] = profit_lost``
Та записуємо у файл "data.csv" для зручності перегляду.

Також рахуємо і виводимо наш загальний прибуток: ``print(f"Profit: {data["Profit/Lost"].sum()}")``

### Знаходження точок купівлі/продажу
Для того, щоб знайти дати та ціну купівлю або продажу якції - ствроюємо нові пусті списки:
```python
buy_dates = []
buy_prices = []
sell_dates = []
sell_prices = []
```
Далі проходимось циклом від другого дня(адже ми порівнюємо дні із попередніми) до останнього:
```python
for i in range(1, len(data)):
```
Так як зміни сигналу з 0 на 1 означає купівлю, додаємо його до списку наступним кодом:
```python
for i in range(1, len(data)):
    if data["Signal"].iloc[i] == 1 and data["Signal"].iloc[i - 1] == 0:
        buy_dates.append(data.index[i])
        buy_prices.append((data["Moving average - 20"].iloc[i] +
                           data["Moving average - 50"].iloc[i]) / 2)
```
Спочатку точки купівлі та продажу розміщувалися на ціні закриття `Close`, оскільки саме вона відповідає ціні купівлі/продажу. 
Для покращення візуалізації ми перенесли точки на середнє значення між `Moving average - 20` та `Moving average - 50`. 
Це дозволило зробити місця перетину ковзних середніх, які подають торговий сигнал, більш точним на графіку.
